# HTML, CSS & JS

:: content ::

<br>
<div class="leading-10">
<span class="font-bold">HTML</span> (HyperText Markup Language): A markup language used to define the structure and semantics of web content using <span class="font-bold">tags</span> and <span class="font-bold">attributes</span>
</div>
<br>
<div class="leading-10">
<span class="font-bold">CSS</span> (Cascading Style Sheets): A stylesheet language used to describe the presentation layer of HTML documents using <span class="font-bold">selectors</span> and <span class="font-bold">properties</span>
</div>
<br>
<div class="leading-10">
<span class="font-bold">JavaScript</span>: A high-level programming language that enables client-side logic on web pages using <span class="font-bold">DOM manipulation</span>, <span class="font-bold">event handling</span>, and <span class="font-bold">API communication</span>.

# Dependency Injection

:: content ::

<img src="./assets/DI.jpg">
<br>

<div class="leading-10">
Design pattern where a class receives its dependencies from an external source rather than creating them itself. Instead of a class being responsible for creating the objects it needs to function, those objects are "injected" into it, typically through its constructor or setter methods.
</div>

# Software Testing

:: content ::

<br>
<div class="leading-10">
<div><span class="font-bold">Unit Testing</span> — Tests indiviual components or modules</div>
<div><span class="font-bold">Integration Testing</span> — Verifies the interactions and interfaces between integrated modules or components</div>
<div><span class="font-bold">Smoke Testing</span> — A quick, high-level test to ensure the most critical functions of a build are working</div>
<div><span class="font-bold">System Testing</span> — Evaluates the complete system to ensure it meets specified requirements and functions as a whole</div>
<div><span class="font-bold">Regression Testing</span> — Ensures that new changes or bug fixes do not negatively impact existing functionality</div>
<div><span class="font-bold">User Acceptance Testing (UAT)</span> — The final stage, where end-users test the software in a real-world</div>
</div>

:: title ::
# Software Testing Techniques

:: content ::

<br>
<div class="leading-10">
<div><span class="font-bold">Black-Box Testing</span> — The tester has no knowledge of the internal code structure and focuses on validating the functionality based on specifications</div>
<br>
<div><span class="font-bold">White-Box Testing</span> — The tester has full access to the source code and uses this knowledge to design test cases that verify the internal logic</div>
<br>
<div><span class="font-bold">Gray-Box Testing</span> — A combination of black-box and white-box testing, where the tester has partial knowledge of the internal structure</div>
</div>
